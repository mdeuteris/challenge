About The Challenge

We want to help an agency configure its users. Create, edit, delete and list. These are the main operations that should excel.

  • First of all you'll declare an instance of docker for Node js. Do not scream just yet! :scream_cat:
  • Secondly, use your tools wisely. Node - Express - Apollo - GraphQl. There are the ones we are really interested. Show off your skills!
  • To store your data which then will be information, Mongo & Mongoose are the way to go.
  • Start implementing your CRUD! Any other service will be highly apreciated.
  • Test your functionalities. You never know when a disaster may happen. If its a feature test, show more than 1. :umbrella:. Test an insertion also. Jest may be your friend.
  • Watch out! We dont want any intruder messing up with your data. Use a JWT for authentication.

Build with

You can decide the technique. Just take a look at these techs to not mislead the challenge objective

Remember to maintain scalability & optimization. Also to have a clear folder structure.

Prerequisites

  1. Mount over Docker for Node js
  2. Clone the repo
    git clone https://github.com/flydevs/challenge-nodejs.git
  3. Enter your API keys in .ENV
    API_KEY = 'ENTER YOUR API';

To Do's

  • At least one feature test
  • Users collection (id,name, lastname, age)
  • Expand collection data with creativity and more functionality!
  • Postman file to test all routes is a MUST. We want to show you part of a daily routine :heart_eyes:
  • A nice Readme. Explain how to run your work. What to expect, project folder organization and most important a brief explanation of flow
  • Filter users by name or lastName
  • Personalized logger
  • Use of Lodash. We love it, we hope you do as well
  • HTTP protocols use wise declarations

To Dont's

  • Large files throwing to garbage readability
  • Weird variable names
  • Recursive functions with no exit condition
  • Unnecessary dependencies
  • No readme for installation
  • Unreadable structures
  • Upload of node modules
  • Initial commit with whole project
  • Weird commits names

Will Be Valued

  • Creativity
  • Performance
  • Error handler middleware
  • Perfect resolvers
  • Optimization
