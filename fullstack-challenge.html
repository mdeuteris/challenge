About The Challenge

We want to develop a web application to search for Pokemons. The solution must meet the following requirements:

  • Free choice of design. Use React for the front, and a back / middleware in nodejs or serverless for communication with the pokeapi.
  • The front end must consume the information from an endpoint in our node api that will act as middleware against the PokeApi (https://pokeapi.co/)
  • At a minimum you have to be able to search for Pokemons by partial name. (Ex: pika)
  • We will value simplicity and neatness. Also added aimed at improving the code quality.
  • Test your functionalities. You never know when a disaster may happen. If its a feature test, show more than 1. :umbrella:. Test an insertion also. Jest may be your friend.
  • Share the solution in a public repo (Github, Bitbucket) with instructions for setup the application in the Readme. Send the repo link to rrhh@flydevs.com

If you have any questions, do not hesitate to contact us at rrhh@flydevs.com

To Do's

  • At least one feature test
  • Pokemons collection (id,name, lastname, age)
  • Expand collection data with creativity and more functionality!
  • Postman file to test all routes is a MUST. We want to show you part of a daily routine :heart_eyes:
  • A nice Readme. Explain how to run your work. What to expect, project folder organization and most important a brief explanation of flow
  • Filter Pokemons by name
  • Use of Lodash. We love it, we hope you do as well
  • HTTP protocols use wise declarations

To Dont's

  • Large files throwing to garbage readability
  • Weird variable names
  • Recursive functions with no exit condition
  • Unnecessary dependencies
  • No readme for installation
  • Unreadable structures
  • Upload of node modules
  • Initial commit with whole project
  • Weird commits names
