About The Challenge

We want to develop a small web app that allows users to visualize movies on a local theatre.

  • First of all we'll display the list of movies following figma design. Data will be loaded from api provided.
  • A search bar is needed. Filter movies and check for unnecessary renders.
  • On click of a movie, movie details will be displayed on a separate page.
  • Users can save favourite movies and these could be shown on a separate page. (This is a plus)
  • You can use any library/framework/plugin you want.
  • Figma design: Design

Build with

You can decide the technique. Just ask for which library/framework the recruitment process expects. If you use React, adding gatsby or Next would be LOVELY.

Prerequisites

  1. Register in rapidapi.com and pick a Public API from here https://rapidapi.com/search/imdb
  2. You created a public repository where your project will work and once it is finished, share the link of it

To Do's

  • At least one unit test
  • Use scss. BEM methodology
  • Constants declaration and exports
  • This design is optimized for 480px. It needs to be responsive up to 1920px. Follow mobile design pattern. Creativity will be appreciated. :heart_eyes:
  • If using React, show off your state management skills! Use a nice context structure or go with Redux.
  • If waiting for a React component to make initial render, wrap it up with SUSPENSE. Show off your UX skills. A spinner will be nice.
  • Upload your project to Netlify, Heroku....

To Dont's

  • Large components
  • Weird variable names
  • Long render times
  • Re-peated css
  • No readme for installation
  • Unreadable structures
  • Upload of node modules
  • Initial commit with whole project
  • Weird commits

Will Be Valued

  • Auto-complete search bar
  • Filmography within movie details
  • Suspense and callbacks
  • Fade in - Fade out cards and text
  • Favorites page
