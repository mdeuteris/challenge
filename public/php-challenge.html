​ We want to help an agency configure its users. create, edit, delete and list. These are the main operations that should excel. ​ First of all you need to know with use Docker, so be prepare to use it. ​

  • Use your tools wisely. PHP, Apache/Nginx & MySql. There are the ones we are really interested.
  • Consider use mailhog (email testing tool for developers)
  • Start implementing your CRUD as a RESTful API!
  • Test your functionalities.
  • Watch out! We don't want any intruder messing up with your data. Use a JWT for authentication. ​

Getting Started

Although you may be asked many things, we want you to know the fact that we appreciate a lot your effort! We know you may not have lots of time to spare, so we’ll value every piece of code you write. Take this in consideration! ​ We don't want to buy any NASA’s computer to launch your work. SO PLEASE! Check for load effort. ​

To Do’s

  • Test your features
  • Postman file to test all routes is a MUST. We want to show you part of a daily routine
  • A nice Readme. Explain how to run your work. What to expect, project folder organization and most important a brief explanation of flow
  • Filter users by name or lastName ​

To Dont’s

  • Large files throwing to garbage readability
  • Weird variable names
  • Recursive functions with no exit condition
  • Unnecessary dependencies
  • No readme for installation
  • Unreadable structures
  • Upload modules
  • Initial commit with whole project
  • Weird commits names ​

Will Be Valued

  • Creativity
  • Performance
  • Error handler middleware
  • Optimization ​
